# Драг-рейсинг — таблица для ревью (MVP v2)

> Цель документа: единая редактируемая спецификация режима — от названия и экономики до вероятностей каждой ветки каждого ивента.

## 1) Карточка режима

| Поле | Значение (текущий драфт) | Что можно править |
|---|---|---|
| Финальное название | Драг-рейсинг | Нет |
| Формат | Single-player PVE (игрок против системы) | Частично |
| Ключевая петля | Выбор сложности + взноса → выборы на ивентах → финиш/поражение → награда | Частично |
| Валюта входа | Рубины (stake) | Да |
| Рекоменд. длительность забега | 60–150 секунд | Да |
| Цель MVP | Быстрый, мемный, честный по вероятностям режим с высокой реиграбельностью | Нет |

## 2) Сложность: выбор игрока + влияние взноса

> По требованию: **сложность выбирает игрок**, но в коде дополнительно учитываем влияние суммы взноса.

### 2.1 Базовая сложность (что выбирает игрок)

| Difficulty | Ивентов N | Базовый множитель финиша | Целевой Win Rate |
|---|---:|---:|---:|
| EASY | 4 | 1.55 | 58–65% |
| NORMAL | 5 | 1.75 | 45–55% |
| HARD | 6 | 2.10 | 32–42% |
| EXTREME | 7 | 2.60 | 22–30% |

### 2.2 Скрытая прибавка сложности от взноса (stake pressure)

| Stake band | Диапазон рубинов | stake_pressure_penalty |
|---|---:|---:|
| S0 | 10–49 | 0.00 |
| S1 | 50–149 | 0.03 |
| S2 | 150–299 | 0.06 |
| S3 | 300+ | 0.10 |

### 2.3 Формулы MVP

| Метрика | Формула |
|---|---|
| База награды | `base_reward = stake * difficulty_base_multiplier` |
| Эффективный шанс ветки | `effective_success = clamp(base_success - stake_pressure_penalty * stake_pressure_coeff, 0.05, 0.99)` |
| Эффективный шанс фатала | `effective_fatal = clamp(base_fatal + stake_pressure_penalty * fatal_pressure_coeff, 0.00, 0.95)` |
| Итог награды | `final_reward = base_reward * product(event_modifiers)` |

## 3) Подушки безопасности (новое правило)

| Опция | Цена | Эффект |
|---|---:|---|
| Airbag x1 за рубины | 100 рубинов | 1 фатал конвертируется в тяжёлый штраф (без завершения забега) |
| Airbag x1 за бустеры | 5 бесплатных бустеров | То же |
| Ограничение MVP | максимум 1 Airbag на старт забега | Упрощает баланс |

> Покупка Airbag происходит на старте матча до первого ивента.

## 4) Общая платформа для REGULAR/BATTLE/DRAG_RACING

По требованию вводим общий базовый класс/контракт, чтобы общие расчёты (например, суммы голосов/ставок/пот-логики) были в одном месте.

### 4.1 Предлагаемый общий контракт

| Класс/интерфейс | Назначение |
|---|---|
| `AbstractMatchModeEngine` | Общие инварианты: валидировать ставку, считать pot/голоса, общую комиссию/награду, общие проверки статуса |
| `RegularMatchEngine` | Специфика обычной гонки |
| `BattleMatchEngine` | Специфика батлов (создатель, участники, join/start) |
| `DragRacingEngine` | Специфика ивентов, веток, дебафов, airbag |

### 4.2 Общие методы (ядро)

| Метод | Для чего общий |
|---|---|
| `validateStake(...)` | Единая валидация входных сумм |
| `computeCommonPot(...)` | Единый расчёт суммы пула/голосов |
| `applyCommonFee(...)` | Единая комиссия/сервисный сбор |
| `finalizePayout(...)` | Единый базовый механизм выплаты |

## 6) API-контракт (MVP)

| Endpoint | Метод | Назначение | Минимальные поля |
|---|---|---|---|
| `/drag-racing/start` | POST | Старт забега | `stake`, `difficulty`, `buyAirbagBy`(`RUBIES`/`FREE_BUSTS`/`NONE`) |
| `/drag-racing/state` | GET | Текущее состояние | `runId` |
| `/drag-racing/choice` | POST | Применить выбор | `runId`, `eventId`, `branchId` |
| `/drag-racing/result` | GET | Результат забега | `runId` |
